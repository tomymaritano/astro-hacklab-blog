Cómo Construí el Sitio Web de Blacro Studio con Next.js, Tailwind CSS y Framer Motion

Aquí está el proceso de crear un sitio minimalista y animado para Blacro Studio — una agencia creativa — usando Next.js, Tailwind CSS y Framer Motion.

📅 6/23/2025

Cuando el equipo de Blacro Studio me contactó, supe que esto iba a ser más que otro sitio web. Ya tenían una identidad creativa fuerte — mi trabajo era traducir eso en una experiencia web fluida y de alto rendimiento.

Aquí está mi proceso, las herramientas que elegí y lo que aprendí en el camino.

El Brief: Minimalismo con Personalidad

Blacro Studio no quería un sitio web típico de agencia. Querían algo que:

  • Reflejara su filosofía minimalista pero con toques de personalidad únicos
  • Cargara súper rápido — nada de animaciones pesadas que arruinen la UX
  • Se sintiera premium sin ser pretencioso
  • Funcionara perfectamente en móvil (donde está el 70% de su tráfico)

Stack Tecnológico Elegido

Frontend

  • Next.js 14 (App Router para mejor performance)
  • Tailwind CSS (rapid prototyping + design system consistente)
  • Framer Motion (animaciones fluidas y controladas)
  • TypeScript (porque los errores en runtime son caros)

Performance & SEO

  • Vercel para hosting (edge functions + CDN global)
  • Next/Image para optimización automática
  • next-sitemap para SEO automático
  • Lighthouse CI en el pipeline para QA

Proceso de Desarrollo

1. Wireframing & Design System

Antes de código, establecimos:

/* Variables de diseño */
:root {
  --primary: #1a1a1a;
  --accent: #6366f1;
  --text-light: #64748b;
  --spacing-unit: 1.5rem;
}

2. Componentes Base

Crear componentes reutilizables fue clave:

// Button component con variants
const Button = ({ variant = 'primary', children, ...props }) => {
  const baseStyles = "px-6 py-3 font-medium transition-all duration-200"
  const variants = {
    primary: "bg-accent text-white hover:bg-accent/90",
    outline: "border-2 border-accent text-accent hover:bg-accent hover:text-white"
  }
  
  return (
    <button className={`${baseStyles} ${variants[variant]}`} {...props}>
      {children}
    </button>
  )
}

3. Animaciones Thoughtful

Las animaciones debían mejorar la experiencia, no distraer:

// Hero section con animación de entrada
const Hero = () => {
  return (
    <motion.section
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.6, ease: "easeOut" }}
    >
      <h1>Beautiful Digital Experiences</h1>
    </motion.section>
  )
}

Desafíos & Soluciones

Challenge 1: Animaciones Complejas sin Lag

Solución: Usar transform y opacity en lugar de width/height. Framer Motion + will-change CSS.

Challenge 2: Imágenes de Portfolio Pesadas

Solución: Next/Image con placeholder="blur" + WebP automático.

Challenge 3: SEO + Single Page App

Solución: Next.js SSG para páginas estáticas, ISR para contenido dinámico.

Lo Que Aprendí

1. Less is More (Really)

Cada animación, cada color, cada sombra tiene que tener un propósito. El minimalismo no es aburrido si está bien ejecutado.

2. Performance Budget is Sacred

Establecimos un budget de 500kb para el bundle inicial. Toda feature tenía que justificar su peso.

3. Mobile UX ≠ Desktop UX Shrunk

Las interacciones táctiles requieren thinking diferente. Los hover states son irrelevantes.
